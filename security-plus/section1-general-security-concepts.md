## 1.1 Security Controls
**Video length: 11:48**

### Key Concepts
- Control Types:
    - Technical - OS Controls
    - Managerial - Secuirty policy, company rules, standards
    - Operational - Personel rules, profesional development, guards
    - Physical - Limits physical access, locks, card readers
- Usages
    - Preventative - Works *before* to avoid anything happining from step 1 by limiting access
    - Deterrant - Works *before* to cause aprehension twards malicious behavior 
    - Detective - Works *durring* to find any current exploits / attack attempts
    - Corrective - Works *after* to fix any mistakes, reverse impact of attacks, reduce downtime
    - Directive - Works *in present/future* to inform and prevent through teaching information 


### Notes
| Control Types | Preventative | Deterrent | Detective | Corrective | Compensating | Directive |
|---|---|---|---|---|---|---|

| **Managerial** | Security Policy | Demotion | Review Log-in Reports | Security Alert | Separate Duties | Compliance Policy |
| **Operational** | Guard Shack | Reception Desk | Property Patrol | Law Enforcement | Policy Training | |

### The CIA Triad
**Video length: 5:17**

### Key Concepts
- Confidentiality - Make sure only those who should access material can access it (Access Controlls, Encryption, Authention)
- Integrity - Information stays intact when sending/reciving (Hashing, Signatures, Certificates, Non-Repudiation + Proof of Identity)
- Accesability - System must always be up and running so those who need to can access it (Redudancy, Fault Tollorance, Patching)

### Notes
- Note that one of the bigest points of Cybersecurity is making sure theres a good combination of all 3 of the CIA Triad which ensures peak data security

### Non-repudiation
**Video length: 7:57**

### Key Concepts
- Non-Repudiatiion - Ensures proof of sender, and proof of intergrity of the information sent
- Hash - A short string of characters used to represent exact data
- Private Key - Only known to the sender
- Public Key - Known to all
- Digital Siguature - Created with hash of plaintext and private key to be sent with plain text

### Notes
- Works as a signed contract, what you said is what you said
- Proof of intergirty -> Sent = Recived 
- A **hash** can be used, a short string of characters which represents data, can compare hashes to verity data's integrity
- Sending a plaintext with a **digital signature** created with the senders private key, allows the reciver to use the public key to verify integrity and identiy with the hash
